Requirements

Implement a solution that allows us to instantiate a robot, move it around and ask it for its position and direction. Input and output can be very basic, and no visualization is needed. You can even hard code some input and output for various cases.

Implementation:

  • Instantiate Robots by placing a robot prefab in Designer mode.
  • Move robots around by pressing Execute after filling the right-hand text input with either format:
    • "[ground|airborne] <instructions>" to move the first ground or airborne Robot and select* it
    • "<instructions>" to move the selected* Robot
  • Position and compass direction are given for the selected* Robot

* Select a robot by clicking the left-hand buttons or moving it via 2 parameter instructions. The selected Robot will flash yellow and take instructions by default.

Camera Movement:

  • Mouselook is enabled and disabled with Space, allowing you to click and type in the GUI without looking around
  • WSAD and the Arrow Keys both move the character on the X,Z plane
  • In the interests of time and MVP there is no vertical movement
  • Collision detection is disabled for the camera and the Robots

Testing Frameworks Research:

I've never done automated testing in Unity so I researched and attempted some solutions:

  • I tried UUnit but the code was outdated and didn't work properly. I tried to fix it but I don't have the time to get it working, unfortunately.
  • I tried SharpUnit which seemed more well-maintained but doesn't seem to be compatible with UnityScript.
  • "Inversion of Control" may be another viable solution outside the scope of this project.
  • Test Star looks like a great commercial solution and I'd love to have a chance to try it out in the future.

All in all, my impression is that it's a lot easier to do testing if we were to move toward C# instead of UnityScript. Unfortunately, automated testing in Unity is very much in its infancy.
